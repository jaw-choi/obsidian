# 언리얼 리플렉션 (Property System)

언리얼 리플렉션/프로퍼티 시스템은 C++에 없는 런타임 자기정보(타입, 멤버, 함수, 메타데이터)를 엔진이 쓰기 위해 설계된 구조입니다.

## 전체 파이프라인
1. **표시:** 헤더에 `UCLASS`, `UPROPERTY` 등의 매크로로 리플렉션 대상을 표시.
2. **파싱:** UHT(Unreal Header Tool)가 헤더를 파싱하여 `.generated.h` 생성.
3. **등록:** 컴파일 시 생성된 코드가 모듈에 포함되고 런타임에 메타 오브젝트로 등록됨.

## 왜 필요한가?
- **GC(가비지 컬렉션):** `UPROPERTY`로 표시된 참조를 통해 살아있는 객체 추적.
- **직렬화:** 객체 상태를 저장하거나 로드할 때 메타데이터 활용.
- **에디터 노출:** 디테일 패널이나 블루프린트에서 변수를 조작 가능하게 함.

## 주요 매크로
- `UCLASS()`: 클래스 메타데이터 생성.
- `UPROPERTY()`: 멤버 변수 리플렉션 등록.
- `UFUNCTION()`: 함수 리플렉션 등록.
