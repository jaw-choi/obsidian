# CS - HashTable

## 1. 한 줄 요약
HashTable은 키-값 쌍을 저장하고, 키를 사용하여 빠르게 값을 검색할 수 있는 자료구조이다.

## 2. 왜 필요한가? (Why)
- 빠른 데이터 검색 (평균 O(1) 시간 복잡도).
- 대용량 데이터 처리 효율성 및 데이터베이스 인덱싱 활용.

## 3. 핵심 개념 (What)
- 해시 함수(Hash Function)를 사용하여 키를 해시값으로 변환.
- 해시값을 인덱스로 사용하여 배열에 데이터 저장.

## 4. 동작 과정 (How)
1. 키를 해시 함수에 입력하여 해시값 생성.
2. 해시값을 배열의 인덱스로 사용.
3. 해당 인덱스에 키-값 쌍 저장 (충돌 시 Chaining 등 전략 사용).
